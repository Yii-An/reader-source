# 全局规则

## Git Commit 规范

生成 Git commit 消息时，请遵循以下规范：

### 格式

```
<类型>(<范围>): <简短描述>

<详细描述（可选）>

<关联事项（可选）>
```

### 类型（Type）

| 类型       | 说明                                      |
| ---------- | ----------------------------------------- |
| `feat`     | 新功能                                    |
| `fix`      | 修复 Bug                                  |
| `docs`     | 文档变更                                  |
| `style`    | 代码格式调整（不影响代码逻辑）            |
| `refactor` | 代码重构（既不是新功能也不是修复 Bug）    |
| `perf`     | 性能优化                                  |
| `test`     | 添加或修改测试                            |
| `build`    | 构建系统或外部依赖变更（如 npm、webpack） |
| `ci`       | CI 配置文件或脚本变更                     |
| `chore`    | 其他不修改源代码或测试文件的变更          |
| `revert`   | 回滚之前的提交                            |

### 范围（Scope）

范围是可选的，用于说明 commit 影响的模块或组件，例如：

- `api`
- `ui`
- `config`
- `utils`

### 规则

1. **语言**：所有 commit 消息必须使用**中文**
2. **简短描述**：不超过 50 个字符，使用祈使句，首字母无需大写，结尾不加句号
3. **详细描述**：如有必要，在空行后详细说明变更的动机和上下文
4. **关联事项**：如有相关的 Issue 或 PR，在末尾添加引用

### 示例

```
feat(用户模块): 添加用户登录功能

实现了基于 JWT 的用户登录认证机制，包括：
- 登录表单验证
- Token 生成与刷新
- 登录状态持久化

关联 #123
```

```
fix(搜索): 修复搜索结果分页错误

修复了当搜索结果为空时分页组件显示异常的问题
```

```
docs: 更新 README 安装说明
```

```
refactor(解析器): 重构 HTML 解析逻辑

将解析逻辑拆分为独立模块，提高代码可维护性
```
